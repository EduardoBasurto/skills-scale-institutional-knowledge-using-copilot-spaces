# OctoAcme Project Management Docs

Welcome to the centralized index for OctoAcme's project management documentation. This README provides a brief overview of our approach and links to all process documents.

## Project Management Overview

OctoAcme runs projects with the following core principles:

- **Clear roles and responsibilities** — Every project has a named Project Manager, Product Manager, and delivery team with well-defined accountabilities. See [Roles and Personas](octoacme-roles-and-personas.md) for full definitions.
- **Customer-first delivery and measurable outcomes** — We prioritize customer value, define success metrics upfront, and make data-informed decisions throughout the project lifecycle.
- **Iterative planning, delivery, and review cycles** — Work is broken into small, testable increments spanning initiation, planning, execution, release, and retrospective phases.
- **Transparent risk management and open communication** — Risks are tracked in a living Risk Register, and stakeholders receive regular updates through structured communication cadences.
- **Continuous improvement through retrospectives** — After every sprint, release, or milestone we capture what went well, what could be improved, and commit to actionable follow-ups.
